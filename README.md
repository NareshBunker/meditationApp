Exercise 5: Add to Favorites Functionality
User Story Template
Title:
As a user, I want to add an item to my Favorites so that I can save activities or articles I like for quick access later.

Acceptance Criteria:

A heart icon with the text “Add to Favorites” is displayed next to each item; an outlined heart indicates the item is not in Favorites.

Tapping the button adds the item to the Favorites list.

Upon tapping, the button text updates to “Remove from Favorites” and the heart icon changes to filled.

Story Points: 3

User Story Template
Title:
As a user, I want to remove an item from my Favorites so that I can manage my saved content.  

Acceptance Criteria:  

The “Remove from Favorites” button with a filled heart is displayed for items already in Favorites.  

Tapping the button removes the item from the Favorites list and reverts the heart icon to outlined.  

Users can add or remove items anytime, and the UI status updates dynamically.  

Story Points: 2

User Story Template
Title:
As a user, I want a “My Favorites” screen so that I can view and manage all my saved items in one place.  

Acceptance Criteria:  

The “My Favorites” screen displays a list of saved items with their title, category, and duration.  

Users can tap any item to view its detailed screen or start the activity.  

The Favorites list remains organized for easy browsing and quick access.

Story Points: 3

Exercise 6: Daily Reminders
User Story Template
Title:
As a user, I want to view the calendar for the current month and navigate between months so that I can easily select dates for reminders.  

Acceptance Criteria:  

Display the current month with all the days visible.  

Provide navigation arrows to move forward and backward between months.  

Story Points: 3

User Story Template
Title:
As a user, I want to select a date and time for a reminder so that I can schedule it properly.  

Acceptance Criteria:  

Display a default text “Selected Date: None” and “Selected Time: 20:44” to indicate no date has been selected but a default time is chosen.  

Allow users to tap and choose a specific custom time and date.

Story Points: 3

User Story Template
Title:
As a user, I want to add a reminder after selecting a time so that I can schedule it for a future date and time.  

Acceptance Criteria:  

After selecting a time, users can click the “Add Reminder” button to schedule the reminder.  

The system validates that the selected configuration targets a future timestamp.

Story Points: 2

User Story Template
Title:
As a user, I want to see a list of all my reminders so that I can manage them easily.

Acceptance Criteria:

Display a list of all scheduled reminders with their selected date and time.

Provide an option to delete a reminder by clicking a red “Delete” button next to it.

Story Points: 2
